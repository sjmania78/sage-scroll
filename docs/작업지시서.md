# 작업지시서 — Sage Scroll (검증 MVP)

> 프로젝트명 **Sage Scroll** (확정 2026-06-29). 디렉터리/코드네임: `place-atlas`.
> 한 줄: 한국(→아시아) 작가·사상가·철학가 등 지성의 **연고지·생애·저작을 검증된 출처로 지도에 매핑**. **"왜 위대했나"는 다루지 않는다.**
> 모델: **인물 중심 + 생애 지도** (생애 연표 ↔ 지도 장소 연동). 동선 후순위.
> 이름조차 평가어(genius)를 피해 **Sage(현인)+Scroll(두루마리)** 로 정함 — 이름과 정직성을 일치시킴.
> 현재: 한국 지성 6인 적재·생애지도 작동, +5인 수집 중(→11인). 독립 진행(crossstar에 얹지 않음).
> 이 문서는 논의를 거쳐 살아남은 버전이다. **비목표 섹션이 빌드 가드레일**이다.

---

## 0. 한 줄 정의

한국(→아시아) 지성의 **연고 장소·핵심 저작을 검증된 데이터로 지도화**하고, **생애(언제·어디서·무엇을)를 장소에 연결**해 보여주는 사이트. **"왜 위대했나"·"뭘 배울 수 있나"는 다루지 않는다.**

---

## 1. 목표 / 비목표 ← 가장 중요

### 목표
- 인물 → 연고 장소 → **생애 연표(장소 연동)** → 저작 연결
- 검증된 출생지·연고지 좌표 데이터
- 핵심 작품/저서 + **1차 사료 원문 인용** (검증 가능한 사실·전통적 귀속만)

### 비목표 — 빌드 시 절대 생성 금지
- ❌ "왜 특출났나" / 성공 요인 / **성장과정→성과 인과 서사** → **생존편향. 금지.**
- ❌ "이 인물에게 배우는 마스터리·교훈" / **"뭘 배울 수 있는지" AI 정리** → 금지.
- ❌ 현역 인물 사생활·실거주지 → 금지. **사망했거나 공적으로 확립된 연고지만.**
- ❌ 13탭급 대형 구조 → 금지. **이건 검증 MVP다.**
- ❌ AI가 생성한 인물 평가·서술 → 금지. **사실 + 출처만.**

> **위인·지성을 다룰수록 이 가드레일이 더 중요하다.** 연표 항목은 *사실*(출생·이동·관직·저작·사망)만,
> 저작은 *전통적 귀속 + 사료 원문*만. "뭘 배울지"는 사료를 출처와 함께 보여주고 **해석은 독자에게** 맡긴다.
> `works[].note`·`timeline[].event`조차 객관 사실만. (Joe가 5턴 논쟁 끝에 직접 박은 가드레일)

## 2. 범위 (MVP 스코프)

- **대상: 한국 지성** — *외국엔 덜 알려졌지만 한국이 자랑하는* 작가·사상가·철학가 + 국민적 위인. (이 "과소대표" 지점이 유일한 해자)
- **현재 적재(2026-06-29)**: 6인 확정 + 5인 수집 중 = 11인
  - 사상가·철학: 정약용·최한기·신채호 / 시인·작가: 윤동주·이상·한용운·정지용·김유정·이육사 / 국민 위인: 세종대왕·이순신
- **선정 기준**: 검증 가능한 연고지·생애·저작 보유, 공적으로 확립된(사망) 인물.
- **비범위**: 아시아 타국 확장(중국 취푸=공자 등)은 데이터만 보관(`data/raw/qufu.json`), **한국에서 작동·반응 증명이 먼저.** 북한 소재 출생지(김소월·백석 등)는 좌표 확보 어려워 후순위.

## 3. 데이터 모델 ← 진짜 자산

> 노력의 90%를 데이터에 투입. UI 아니다. 검증된 연고지·생애·저작 데이터가 자산, 지도는 껍데기.

### 설계 원칙
1. **장소에 고유 `id`** → 생애 연표(`timeline`)·동선(`routes`)이 장소 id를 참조.
2. **검증 상태 명시** → `verified` + `flags[]`. 미검증을 숨기지 않고 화면 배지로 노출.
3. **좌표 환각 차단** → 공식 출처에 수치 없으면 OSM Nominatim 지오코딩(`coords-osm` flag), 그래도 없으면 `null`+`coords-unverified`. **지어내지 않는다.**
4. **생애 = 사실 연표** → `timeline[].event`는 사실만, 인과·교훈 금지. 가능하면 `place_id`로 지도 연동.
5. **저작 = 사료 인용** → `works[].quote`는 1차 사료 원문, `quote_source`로 출처. AI 해석 없음.

### 인물 레코드 (`public/data/people.json`)
```json
{
  "id": "jeong-yagyong",
  "name_ko": "정약용", "name_en": "Jeong Yak-yong",
  "field": "실학자·문신",
  "birth_year": 1762, "death_year": 1836,
  "birthplace": { "lat": null, "lng": null, "admin": "경기 광주부 마재(현 남양주 조안면)", "source_url": "https://encykorea.aks.ac.kr/...", "source_grade": "A", "flags": ["coords-unverified"] },
  "timeline": [
    { "year": 1801, "event": "신유박해로 강진 유배(18년)", "place_id": "dasan-chodang", "source_url": "...", "source_grade": "A", "flags": [] }
  ],
  "places": [
    { "id": "dasan-chodang", "name": "강진 정약용 유적(다산초당)", "lat": 34.5808, "lng": 126.7450, "type": "유적", "source_url": "...", "source_grade": "A", "flags": ["coords-osm"] }
  ],
  "works": [
    { "title": "목민심서", "year": 1818, "note": "지방 수령 행정 지침서. 강진 유배 중 저술.", "quote": "군자의 학문은 수신이 그 반이요, 나머지 반은 목민이다.", "quote_source": "목민심서 자서", "source_url": "...", "source_grade": "A", "flags": [] }
  ],
  "source_grade": "A", "verified": true, "flags": []
}
```

- `type`: 생가 | 사당 | 묘/능 | 서원 | 학교 | 문학관 | 기념관 | 유적 | 거리 | 기타
- `source_grade`: `A`=공공/공식기관·UNESCO · `B`=백과·학술 · `C`=위키류
- `flags`: `wiki-only` | `coords-unverified` | `coords-osm` | `birthplace-disputed` | `birthplace-presumed` | `overseas` | `authorship-disputed` | `year-approx` | …

### 검증 규칙
- 모든 연고지·연표·사실에 **출처 URL 필수**.
- **위키 단독 = 미검증** → `verified:false`, `flags:["wiki-only"]`, 화면 "미검증" 배지.
- **저작은 전통적 귀속**. 저자 논쟁은 `authorship-disputed` + note. 추정을 사실로 단정 금지.
- **좌표 출처와 사실 출처 분리**: 좌표=OSM 지오코딩(`coords-osm`)이어도 사실(연고지가 거기 있음)은 공식 A급 출처로.

## 4. 기능 (최소)

- **지도 뷰**: 연고지를 장소 마커로 (Leaflet + OSM — 무료)
- **인물 카드**: 생애 연표 + 연고 장소 + 저작(사료 인용) + 출처 링크
- **생애 지도 (주 기능)**: 연표 항목의 📍를 누르면 지도가 그 장소로 이동·강조. *생애를 장소 위에서 따라간다.*
- **동선 뷰 (후순위)**: Joe 지시(2026-06-29) — 여행 동선보다 지도 매핑·인물 깊이에 집중. 데이터 있을 때만 노출.
- 끝. **검색·필터·계정·로그인 = 비범위.**

---

## 5. 기술 제약

- **Simple is best.** 정적 사이트(`public/`, 빌드 없음). 데이터 = JSON 파일. 지도 = Leaflet + OSM(유료 API 금지).
- 로컬 실행: launch.json `place-atlas`(포트 8775) 또는 `cd public && python3 -m http.server`.
- 좌표 보완: OSM Nominatim 지오코딩(공식 출처에 수치 좌표가 없을 때).

## 7. 검증 실험 설계 (빌드 ≠ 목적)

- **측정**: (a) 관심도 — 트래픽·체류·공유 (b) 데이터 구축이 합리적 노력으로 가능한가
- **계속 기준**: 유의미한 유입·반응 또는 문화기관 제휴 가능성.
- **폐기 기준**: N주 무반응 → 버린다. **매몰비용 금지.**
- **차별점**: 위키는 평가·서술이 섞이고 생애지도가 없다. 우리는 *검증된 연고지·생애·사료를 장소 위에 정직하게 통합*한다. + *외국엔 무명인 한국 지성*을 메운다.
